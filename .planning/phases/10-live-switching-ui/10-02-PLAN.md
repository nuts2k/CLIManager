**2b. ProviderTabs.tsx 新增 CLI 独立开关 + Tab 绿色状态点：**

1. 导入新增依赖：
   - `Switch` from `@/components/ui/switch`
   - `Tooltip, TooltipContent, TooltipProvider, TooltipTrigger` from `@/components/ui/tooltip`
   - `useProxyStatus` from `@/hooks/useProxyStatus`
   - `proxyEnable, proxyDisable` from `@/lib/tauri`
2. 在组件内调用 `useProxyStatus()` 获取 `{ proxyStatus }`
3. 获取当前 CLI 的代理状态：`const cliStatus = proxyStatus?.cli_statuses.find(s => s.cli_id === currentCliId)`
4. 计算开关状态和禁用条件：
   - `const globalEnabled = proxyStatus?.global_enabled ?? false`
   - `const cliProxyActive = cliStatus?.active ?? false`（代理实际运行中）
   - `const hasProvider = cliStatus?.has_provider ?? false`（有活跃 Provider）
   - `const switchDisabled = !globalEnabled || !hasProvider`（全局未开启或无 Provider）
   - tooltip 文案：`!globalEnabled ? t("proxy.globalDisabled") : !hasProvider ? t("proxy.noProvider") : ""`
5. `handleCliProxyToggle` 函数：
   - 如果当前 active -> 调用 `proxyDisable(currentCliId)`，成功 toast.success `t("proxy.disableSuccess")`
   - 如果当前非 active -> 调用 `proxyEnable(currentCliId)`，成功 toast.success `t("proxy.enableSuccess")`
   - 失败时解析错误字符串：
     - 端口占用：提取端口号（从 cliStatus?.port 或从错误文本），toast.error `t("proxy.portInUse", { port })`
     - 通用错误：toast.error `t("proxy.enableFailed") + ": " + error`
   - 注意：开关状态由 `proxyStatus` 驱动（通过 proxy-mode-changed 事件自动刷新），无需乐观更新
6. 在 TabsList 和 TabsContent 之间的 `<div>` 行（line 163-175）之后，插入代理开关行：
```tsx
{/* CLI 代理开关行 */}
<div className="flex items-center gap-3 px-1 py-2">
  <span className="text-sm text-muted-foreground">{t("settings.proxyMode")}</span>
  {switchDisabled ? (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger asChild>
          <span>
            <Switch
              checked={cliProxyActive}
              disabled
            />
          </span>
        </TooltipTrigger>
        <TooltipContent>
          <p>{tooltipText}</p>
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  ) : (
    <Switch
      checked={cliProxyActive}
      onCheckedChange={handleCliProxyToggle}
    />
  )}
</div>
```
（将 tooltipText 提取为变量，仅在 switchDisabled 时显示 Tooltip 包装）

7. Tab 标签绿色状态点：修改 TabsTrigger 内容，在每个 tab 的文字后面根据该 CLI 代理是否 active 显示绿色小圆点：
```tsx
<TabsTrigger key={tab.id} value={tab.id}>
```

**注意事项：**

- 端口占用错误提取：cliStatus?.port 已包含端口号，可直接使用
