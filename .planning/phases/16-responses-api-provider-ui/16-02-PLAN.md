<!-- Responses API 非流式响应结构 -->
Responses API 响应示例:
```json
{
  "id": "resp_abc123",
  "object": "response",
  "model": "gpt-4o",
  "output": [
    {
      "id": "msg_xyz", "type": "message", "role": "assistant", "status": "completed",
      "content": [{ "type": "output_text", "text": "Hello!", "annotations": [] }]
    }
  ],
  "usage": { "input_tokens": 10, "output_tokens": 5, "total_tokens": 15 }
}
```

<!-- Responses API SSE 事件流 -->
文本流事件序列: response.created → response.output_item.added → response.content_part.added → response.output_text.delta* → response.output_text.done → response.output_item.done → response.completed

函数调用流事件序列: response.output_item.added(type:function_call, 携带 call_id+name) → response.function_call_arguments.delta* → response.function_call_arguments.done → response.output_item.done → response.completed

       - 输入: output 包含 {type:"function_call", call_id:"call_abc", name:"get_weather", arguments:"{\"location\":\"Tokyo\"}"}
       - 期望: content 包含 {type:"tool_use", id:"call_abc", name:"get_weather", input:{location:"Tokyo"}}
       - 注意: 使用 call_id（不是 id）映射到 tool_use.id

    3. test_mixed_text_and_function_call: 文本 + 工具调用混合响应
       - 输入: output 包含 message(output_text) + function_call

    4. test_usage_passthrough: usage 字段直接透传
       - 输入: usage:{input_tokens:100, output_tokens:50, total_tokens:150}
       - 期望: usage:{input_tokens:100, output_tokens:50}（total_tokens 可省略）

    5. test_stop_reason_inference: stop_reason 推断
       - 有 function_call → stop_reason:"tool_use"
       - 无 function_call + status:completed → stop_reason:"end_turn"
       - status:incomplete → stop_reason:"max_tokens"

    6. test_id_prefix_mapping: id 前缀替换
       - 输入: id:"resp_abc123"
       - 期望: id:"msg_abc123"（resp_ → msg_）

    7. test_error_response_passthrough: 4xx/5xx 错误不经此函数处理（handler 层判断）

    8. test_arguments_deserialize_fallback: arguments 反序列化失败时包装为 {"raw":"原字符串"}
       - 输入: arguments:"invalid json {"
       - 期望: input:{"raw":"invalid json {"}

    9. test_text_stream_sequence: 文本流式事件完整序列
       - 输入: response.created → output_item.added(message) → content_part.added → output_text.delta("Hel") → output_text.delta("lo") → output_text.done → output_item.done → response.completed(usage)
       - 期望: message_start → content_block_start(text,index:0) → content_block_delta(text_delta:"Hel") → content_block_delta(text_delta:"lo") → content_block_stop → message_delta(stop_reason:end_turn,usage) → message_stop

    12. test_stream_no_deferred_start: 无 Deferred Start 机制
