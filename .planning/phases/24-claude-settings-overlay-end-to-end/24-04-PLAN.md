<task type="auto">
  <name>Task 1: 后端实现 apply 命令 + 保存后立即 apply + 启动 best-effort apply</name>
  <files>src-tauri/src/commands/claude_settings.rs, src-tauri/src/commands/mod.rs, src-tauri/src/lib.rs, src-tauri/src/storage/icloud.rs</files>
  <action>
在 `src-tauri/src/commands/claude_settings.rs` 增加 apply 能力，并把“错误可见性”做成事件：

1) 定义事件名（保持稳定、便于前端监听）：
- 成功：`claude-overlay-apply-success`
- 失败：`claude-overlay-apply-failed`
- 保护字段被忽略提示：`claude-overlay-protected-fields-ignored`

2) 新增 `#[derive(Serialize)]` 的 payload：
- success: { source: "save"|"startup"|"watcher", settings_path: string }
- failed: { source, error: string }
- ignored: { source, paths: string[] }

3) 新增 `#[tauri::command] pub fn apply_claude_settings_overlay(app_handle: tauri::AppHandle, source: String) -> Result<(), AppError>`：
- 读取当前 local settings -> 获取 claude 的 active provider
  - 若有活跃 provider：用 ClaudeAdapter.patch(provider) 执行（保证保护字段最终值由 provider/proxy 决定）
  - 若无活跃 provider：仍允许 apply overlay，但必须剥离 overlay 内保护字段，避免写入错误 token/base_url（COVL-07）
- 读取 overlay 文件（Option<String>）
  - None：视为 noop（best-effort），不报错
  - Some(text)：必须是合法 JSON 且 root object；否则 Err(AppError::Validation(...)) + emit failed（COVL-12）
- 当 overlay 中包含保护字段：strip 并 emit ignored 事件（COVL-08）
- 当 `~/.claude/settings.json` 存在但非法 JSON：保持现有策略，拒绝写入并返回 Validation（COVL-13）

4) 修改 set_claude_settings_overlay：保存成功后立即调用 apply（source="save"），若 apply 失败：
- set 命令整体返回 Err（强一致，COVL-09）
- 同时 emit failed 事件让 UI 可见

5) 启动 best-effort apply：在 `src-tauri/src/lib.rs` 的 `.setup()` 阶段追加：
- spawn 一个 async 任务调用 apply（source="startup"）
- 失败只 log + emit failed，不阻断启动（COVL-10）

6) 注册命令：在 invoke_handler 加上 `commands::claude_settings::apply_claude_settings_overlay`

注意：
- 保存后立即 apply 属于强一致：失败必须让前端 invoke 报错
- startup/watcher 属于 best-effort：失败只需要可见（事件 + log），不应 panic
</action>
