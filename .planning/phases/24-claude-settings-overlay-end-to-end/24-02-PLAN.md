- `type ClaudeSettingsOverlayStorage = { location: "icloud" | "local_fallback"; file_path: string; sync_enabled: boolean; }`
- `type ClaudeSettingsOverlayState = { overlay_json: string | null; storage: ClaudeSettingsOverlayStorage }`

- `setClaudeSettingsOverlay(overlayJson: string): Promise<{ storage: ClaudeSettingsOverlayStorage }>` → `invoke("set_claude_settings_overlay", { overlayJson })`

注意：
- 这里不做 JSON 校验逻辑，只做类型与调用封装；校验由 UI + 后端共同负责
- 命令名必须与 24-01 保持一致，供后续 SettingsPage 直接调用

<task type="auto">
  <name>Task 2: 在 SettingsPage 实现 Claude overlay 读取/编辑/保存 UI，并补齐中英文文案</name>
  <files>src/components/settings/SettingsPage.tsx, src/i18n/locales/zh.json, src/i18n/locales/en.json</files>
  <action>
在 `Settings → Advanced` 中增加一个可执行的 Claude overlay 小节，必须直接实现以下链路，而不是停留在占位文案：

1) 首次加载：
- 在 `SettingsPage.tsx` 为 Claude overlay 新增本地状态：`overlayJson`、`initialLoading`、`isSaving`、`storageInfo`、`loadError`、`saveError`
- 组件首次进入时（`useEffect`）调用 `getClaudeSettingsOverlay()`：
  - 将返回的 `overlay_json` 回填到多行输入框；`null` 时显示为空字符串
  - 将 `storage.location`、`storage.file_path`、`storage.sync_enabled` 写入 UI 状态
  - 若读取失败，在 Claude 小节内显示明确错误文案，不要只 `console.error`

2) 可编辑多行 JSON：
- 使用 Task 1 的 `Textarea` 做受控输入，内容绑定 `overlayJson`
- 文本框支持多行 JSON 编辑，保留用户输入，不要在输入过程中自动格式化
- 提供保存按钮；保存中按钮禁用，并显示 saving/loading 状态

3) 保存流程：
- 点击保存时先在前端做一次 JSON 校验：
  - 非法 JSON → 直接在 Claude 小节显示验证错误，不调用后端
  - 合法 JSON 但 root 非 object → 同样显示错误并拒绝保存
- 前端校验通过后调用 `setClaudeSettingsOverlay(overlayJson)`
- 保存成功后，立即再次调用 `getClaudeSettingsOverlay()` 刷新：
  - 回填后端 pretty 化后的最终文本
  - 更新 storageInfo（location/file_path/sync_enabled）
  - 清除旧错误状态
- 保存失败时：
  - 保留用户当前输入，不要清空文本框
  - 在 Claude 小节内显示可见的保存错误/校验错误（建议 inline error block，不能只靠 toast）
  - 如需要可额外 toast，但 inline 可见错误是必需项

4) 存储位置与保护字段提示：
- 在小节中显式展示：
  - 存储位置：iCloud / 本地降级
  - 文件路径：`storage.file_path`
  - 是否跨设备同步：根据 `sync_enabled` 展示“是/否”或等价文案
- 在编辑器附近放置清晰说明：
  - `env.ANTHROPIC_AUTH_TOKEN`
  - `env.ANTHROPIC_BASE_URL`
  由 Provider/Proxy 管理，overlay 中即使填写也不会生效

5) i18n：
- 在 `src/i18n/locales/zh.json` 与 `src/i18n/locales/en.json` 增加 Claude overlay 所有文案 key
- 文案至少覆盖：标题、说明、编辑器 label、占位文本、保存按钮、加载中/保存中、存储位置、文件路径、同步状态、保护字段说明、JSON 校验错误、保存失败、读取失败、成功提示
- 不要把新字符串硬编码在组件里
