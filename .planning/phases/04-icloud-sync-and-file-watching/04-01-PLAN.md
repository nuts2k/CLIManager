<objective>
Create the FSEvents-based file watcher backend that monitors the iCloud providers directory for changes from other devices, filters and debounces events, tracks self-writes, and emits Tauri events to the frontend. When active provider files change, automatically re-patch CLI configs.

Purpose: This is the core sync engine -- without it, provider changes from other devices are invisible until app restart.
Output: Watcher module with self-write tracking, integrated into Tauri app lifecycle and provider commands.
</objective>

  <name>Task 1: Create watcher module with self-write tracker and event processing</name>
  <files>src-tauri/Cargo.toml, src-tauri/src/watcher/mod.rs, src-tauri/src/watcher/self_write.rs</files>
  <behavior>
    - SelfWriteTracker::record_write(path) stores path with current timestamp
    - SelfWriteTracker::is_self_write(path) returns true if path was written within last 1 second
    - SelfWriteTracker::is_self_write(path) returns false if path was written more than 1 second ago
    - SelfWriteTracker::is_self_write(path) returns false for unrecorded paths
    - SelfWriteTracker::is_self_write cleans up expired entries
    - Event filtering: only .json extension files pass through
    - Event filtering: .icloud, .tmp, and other non-.json files are ignored
    - Event deduplication: multiple events for same file stem are deduplicated in a batch
  </behavior>
  <action>
    1. Add dependencies to Cargo.toml:
       ```toml
       notify = { version = "8", features = ["macos_fsevent"] }
       notify-debouncer-mini = { version = "0.7", features = ["macos_fsevent"] }
       ```

    2. Create `src-tauri/src/watcher/self_write.rs`:
       - `SelfWriteTracker` struct with `writes: Mutex<HashMap<PathBuf, Instant>>`
       - `new()` constructor
       - `record_write(&self, path: PathBuf)` -- inserts path + Instant::now()
       - `is_self_write(&self, path: &PathBuf) -> bool` -- checks if path exists with timestamp < 1 second; removes expired entries
       - Unit tests for all behaviors listed above (use `std::thread::sleep(Duration::from_millis(1100))` for expiry tests)

    3. Create `src-tauri/src/watcher/mod.rs`:
       - `mod self_write; pub use self_write::SelfWriteTracker;`
       - `#[derive(Clone, serde::Serialize)] pub struct ProvidersChangedPayload { pub changed_files: Vec<String> }`
       - `pub fn start_file_watcher(app_handle: AppHandle) -> Result<(), Box<dyn std::error::Error>>`:
         - Call `crate::storage::icloud::get_icloud_providers_dir()` to get watch path (ensures dir exists)
         - Create `notify_debouncer_mini::new_debouncer` with 500ms duration
         - In the debouncer callback: call `process_events(events, &handle)`
         - Watch the providers_dir with `RecursiveMode::NonRecursive`
         - Use `std::mem::forget(debouncer)` to keep it alive for app lifetime
       - `fn process_events(events: Vec<DebouncedEvent>, app_handle: &AppHandle)`:
         - Get `SelfWriteTracker` from `app_handle.state::<SelfWriteTracker>()`
         - Filter events: only `.json` extension AND NOT self-write
         - Extract file stems, deduplicate via `HashSet`
         - If changed_files not empty:
           - Call `crate::commands::provider::sync_active_providers()` for auto re-patch. If it fails, log error and emit "sync-repatch-failed" event with error string
           - Emit "providers-changed" event with `ProvidersChangedPayload { changed_files }`
       - `use tauri::{AppHandle, Emitter, Manager};` for emit and state access
       - Add `#[cfg(test)]` module with a test for the filter/dedup logic (extract filtering into a pure testable function)
  </action>
