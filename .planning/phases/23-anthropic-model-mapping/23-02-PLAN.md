修改 ProviderDialog.tsx，让 Anthropic 协议也显示模型映射区域，但字段全部可选、无 placeholder：

**1. 修改 showModelMapping 条件（第 221-223 行）:**
改为始终显示（所有协议都显示模型映射）：
```typescript
const showModelMapping = true;
```
或者也可以写为 `form.protocolType === "anthropic" || form.protocolType === "open_ai_chat_completions" || form.protocolType === "open_ai_responses"`，但既然三种协议都要显示，直接用 `true` 更简洁。

**2. 修改 handleSave 中的 upstreamModel 必填校验（第 246-252 行）:**
Anthropic 协议时 upstreamModel 不做必填校验，只有 OpenAI 系列才要求必填：
```typescript
const isOpenAiProtocol = form.protocolType === "open_ai_chat_completions"
  || form.protocolType === "open_ai_responses";
if (isOpenAiProtocol && form.upstreamModel.trim().length === 0) {
  setErrors((prev) => ({
    ...prev,
    upstreamModel: t("validation.upstreamModelRequired"),
  }));
  return;
}
```

**3. 修改 upstreamModel 输入框的 placeholder（第 388-389 行）:**
Anthropic 协议时无 placeholder（空字符串），OpenAI 系列保持显示建议值：
```tsx
placeholder={
  form.protocolType === "anthropic"
    ? ""
    : getSuggestedUpstreamModel(form.protocolType)
}
```
注意：`getSuggestedUpstreamModel("anthropic")` 已经返回 `""`，所以也可以保持 `placeholder={getSuggestedUpstreamModel(form.protocolType)}` 不改。但为了明确表达「Anthropic 无建议值」的意图，推荐显式处理。

**4. 修改 updateProtocolType 中的 Anthropic 特殊处理（第 173-178 行）:**
当从 OpenAI 切换到 Anthropic 时，不再强制清空 upstreamModel。改为：如果用户已手动填写则保留，否则按建议值填充（Anthropic 的建议值本来就是空字符串，所以效果等价）：
```typescript
upstreamModel: shouldResetUpstreamModel
  ? nextSuggestedUpstreamModel
  : prev.upstreamModel,
```
这样 Anthropic 和 OpenAI 走同一个分支逻辑，`getSuggestedUpstreamModel("anthropic")` 返回 `""` 自然处理了默认空值。

**5. 模型映射对的 placeholder（第 411-424 行）:**
当前 placeholder 使用 `t("provider.sourceModel")` 和 `t("provider.targetModel")`。对 Anthropic 协议同样适用（例如 source="claude-sonnet-4-6", target="my-custom-sonnet"），保持不变即可。

注意事项：
- 不要改动 ProviderTabs.tsx 中 handleSave 的 upstream 字段处理逻辑，它已经正确处理了空字符串 -> null 的转换
- 不要改动 i18n locale 文件，现有翻译 key 已足够
- 编辑已有 Anthropic Provider 时，如果之前没有 upstream_model 和 upstream_model_map 数据，打开编辑对话框应显示空的映射区域（不要填充任何建议值）
