重构 ProviderCard.tsx，实现两大改造：

**A. 操作按钮外露（HOME-01）：**

1. 新增 import：从 lucide-react 引入 Pencil, Copy, Play, Trash2, MoreVertical（保留）；从 @/components/ui/tooltip 引入 Tooltip 四件套

2. 将现有的 `{/* Action buttons (hover reveal) */}` 区域从 "hover 显隐的 Switch + DropdownMenu" 重构为 "始终可见的图标按钮组"：

   - 移除外层 `opacity-0 group-hover:opacity-100` 类名，改为 `flex items-center gap-0.5`（按钮始终可见）
   - 保留 Loader2 旋转图标（isLoading 时显示）

3. 按钮排列顺序（从左到右）：
   - 切换按钮（非活跃卡片时显示）：保持已有 Switch Button，但改为 `variant="ghost" size="icon-sm"` 的图标按钮，使用 `ArrowRightLeft` 或 `ToggleRight` 图标（从 lucide-react），加 Tooltip 显示 t("actions.switch")
   - 编辑按钮：`variant="ghost" size="icon-sm"`，图标 `Pencil className="size-4"`，Tooltip 显示 t("actions.edit")
   - 复制按钮：`variant="ghost" size="icon-sm"`，图标 `Copy className="size-4"`，Tooltip 显示 t("actions.copy")
   - 测试按钮：`variant="ghost" size="icon-sm"`，图标 `Play className="size-4"`，Tooltip 显示 t("actions.test")
   - 删除按钮：`variant="ghost" size="icon-sm"`，图标 `Trash2 className="size-4"`，Tooltip 显示 t("actions.delete")。添加 `hover:text-destructive` 类名实现 hover 变红
   - DropdownMenu（仅含"复制到"子菜单）：保留 MoreVertical 图标按钮，但 DropdownMenuContent 中只保留 copyTo 相关的 DropdownMenuSub。如果 otherClis.length === 0 则不渲染这个 DropdownMenu

4. 每个图标按钮用 TooltipProvider > Tooltip > TooltipTrigger(asChild) + TooltipContent 包裹。注意：可以在整个操作区域外层放一个 `<TooltipProvider delayDuration={300}>` 包裹所有按钮，避免重复嵌套

5. 所有操作按钮都加 `disabled={isLoading}` 属性

**B. 卡片 hover 升起效果（HOME-02）：**

6. 修改卡片最外层 div 的 className：
   - 共有类（活跃和非活跃）：添加 `transition-all duration-200 hover:shadow-md hover:-translate-y-0.5`
   - 将现有的 `transition-colors` 改为 `transition-all duration-200`（包含 shadow、transform、border-color、background 过渡）
   - 非活跃卡片 hover：保持现有 `hover:border-border/80 hover:bg-accent/30`，新增 `hover:shadow-md hover:-translate-y-0.5`
   - 活跃卡片 hover：保持橙色边框 `border-status-active/50 bg-status-active/5`，同样加 `hover:shadow-md hover:-translate-y-0.5`，hover 时边框可以加强为 `hover:border-status-active/70`

7. 添加 `shadow-sm` 作为卡片默认阴影（从 shadow-sm 过渡到 hover:shadow-md），确保 hover 前后阴影变化可感知
