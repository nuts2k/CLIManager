    - scan_codex_config reads ~/.codex/config.toml for base_url with provider-scoped fallback: check model_provider field -> model_providers.<active>.base_url -> top-level base_url -> empty string
    - scan_codex_config returns has_api_key=false when OPENAI_API_KEY is missing or empty
    - scan_cli_configs returns empty vec when neither ~/.claude/ nor ~/.codex/ config dirs exist (no error)

  <action>
Create `src-tauri/src/commands/onboarding.rs` with:

1. `DetectedCliConfig` struct (Serialize):
   - `cli_id: String` ("claude" or "codex")
   - `cli_name: String` ("Claude Code" or "Codex")
   - `api_key: String` (full key, masking done in frontend)
   - `base_url: String`
   - `protocol_type: ProtocolType` (from crate::provider)
   - `has_api_key: bool`

2. Internal `scan_claude_config_in(home_dir: &Path) -> Option<DetectedCliConfig>`:
   - Read `{home_dir}/.claude/settings.json`
   - Parse with serde_json::Value
   - Extract `env.ANTHROPIC_AUTH_TOKEN` (default empty string)
   - Extract `env.ANTHROPIC_BASE_URL` (default "https://api.anthropic.com")
   - Return None if file doesn't exist or JSON parse fails (log::warn on parse failure)
   - Set `has_api_key = !api_key.is_empty()`

3. Internal `scan_codex_config_in(home_dir: &Path) -> Option<DetectedCliConfig>`:
   - Read `{home_dir}/.codex/auth.json` for OPENAI_API_KEY
   - Read `{home_dir}/.codex/config.toml` for base_url using toml_edit:
     - Check `model_provider` field value
     - If found, look for `model_providers.<active>.base_url`
     - Fall back to top-level `base_url`
     - Default to empty string if not found
   - Return None if neither auth.json nor config.toml exists
   - Return Some even if only one file exists (partial config)
   - log::warn on parse failures, skip gracefully
   - Set `has_api_key = !api_key.is_empty()`

4. `#[tauri::command] pub fn scan_cli_configs() -> Result<Vec<DetectedCliConfig>, AppError>`:
   - Get home dir via dirs::home_dir()
   - Call scan_claude_config_in and scan_codex_config_in
   - Collect non-None results into vec
   - Return Ok(results) -- never returns Err, edge cases produce empty vec

5. Register in `src-tauri/src/commands/mod.rs`: add `pub mod onboarding;`

6. Register in `src-tauri/src/lib.rs`: add `commands::onboarding::scan_cli_configs` to generate_handler

7. Tests use `_in` variants with tempdir for isolation (consistent with project pattern):
   - `test_scan_claude_valid_config` -- full settings.json with env block
   - `test_scan_claude_missing_api_key` -- settings.json exists but no ANTHROPIC_AUTH_TOKEN -> has_api_key=false
   - `test_scan_claude_missing_file` -- no settings.json -> None
   - `test_scan_claude_corrupted_json` -- invalid JSON -> None (no panic)
   - `test_scan_codex_valid_config` -- both auth.json and config.toml present
   - `test_scan_codex_provider_scoped_base_url` -- config.toml with model_providers section
   - `test_scan_codex_missing_files` -- no codex dir -> None
   - `test_scan_codex_partial_config` -- only auth.json, no config.toml -> still returns result
   - `test_scan_codex_corrupted_toml` -- invalid TOML -> still returns result from auth.json
