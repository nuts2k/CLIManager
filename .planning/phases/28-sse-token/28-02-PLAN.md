**3. 日志采集改造——流式请求获取 row_id + spawn 后台 task：**

修改日志采集块（约第 709-738 行），改为：
```rust
// 日志采集
let mut log_row_id: Option<i64> = None;
if let Some(tx) = state.log_sender() {
    let entry = LogEntry {
        // ... 现有字段 ...
        ttfb_ms: Some(ttfb_ms),  // 统一填充
        duration_ms: if upstream_is_sse {
            None  // 流式请求 duration 在 EOF 后由后台 task 填充
        } else {

    };

        Ok(()) => {

            // 注意：try_send 后 log_worker 异步执行 INSERT 并返回 id
            // 但这里无法同步获取 id——需要改用不同方式
        }
        Err(e) => log::warn!("日志 channel 发送失败（可能已满）: {}", e),
    }
}
```

**关键设计问题：row_id 获取方式。** 当前 `try_send` 是 fire-and-forget，handler 无法获得 INSERT 后的 rowid。有两个方案：

**方案 A（推荐——后台 task 直接执行 INSERT+UPDATE）：** 流式请求不走 log_worker 的 try_send，而是在后台 task 中自行执行 INSERT（等 stream EOF 后填充 token 再 INSERT 完整记录）。这样无需 UPDATE，一次 INSERT 即包含完整数据。

但这改变了设计意图（CONTEXT.md 说"Phase 27 已为流式请求写入基础元数据（token/duration/ttfb 留 null），Phase 28 在 EOF 后 UPDATE 同一行"），且 log_worker 的 emit type="new" 不会发生。

**方案 B（推荐——后台 task 自行 INSERT 并 emit new+update）：** 流式请求的日志不通过 log_worker，而是在 handler 返回响应后由后台 task 统一处理：
1. 先 INSERT 基础元数据（token=None），获得 rowid，emit type="new"
2. 等待 stream EOF（rx.await）
3. UPDATE 填充 token/duration，emit type="update"

这保持了 CONTEXT.md 的设计意图。

**方案 C（最简——在 handler 中直接 INSERT 并获取 rowid）：** 流式请求在 handler 中直接调用 `db.insert_request_log()` 而不走 log_worker channel，这样可以同步获取 rowid。然后 spawn 后台 task 等待 rx 并 UPDATE。

**采用方案 C**——最简单且直接解决问题。流式请求跳过 log_worker channel，直接调用 DB INSERT：

```rust
if upstream_is_sse {
    // 流式请求：直接 INSERT 获取 rowid（不走 log_worker channel）
    if let Some(app_handle) = state.app_handle() {
        use tauri::{Emitter, Manager};
        if let Some(db) = app_handle.try_state::<crate::traffic::TrafficDb>() {
            match db.insert_request_log(&entry) {
                Ok(id) => {
                    // emit type="new"（token 为 None 的初始状态）
                    let payload = crate::traffic::log::TrafficLogPayload::from_entry(id, &entry, "new");
                    let _ = app_handle.emit("traffic-log", &payload);
                    log_row_id = Some(id);
                }
                Err(e) => log::warn!("流式日志 INSERT 失败: {}", e),
            }
        }
    }
} else {
    // 非流式请求：走 log_worker channel（现有逻辑）
    if let Some(tx) = state.log_sender() {
        if let Err(e) = tx.try_send(entry) {
            log::warn!("日志 channel 发送失败（可能已满）: {}", e);
        }
    }
}
```

**4. spawn 后台 task 等待 stream EOF：**

在日志采集之后、return builder.body(body) 之前：
```rust
// 流式请求：spawn 后台 task 等待 stream EOF 后 UPDATE token
if let (Some(rx), Some(row_id)) = (streaming_token_rx, log_row_id) {
    let app_handle = state.app_handle().cloned();
    let start_time_clone = start_time;
    let ttfb = ttfb_ms;
    let entry_for_update = entry.clone();  // 用于构建 update payload

    tokio::spawn(async move {
        match rx.await {
            Ok(token_data) => {
                let duration_ms = start_time_clone.elapsed().as_millis() as i64;
                if let Some(ref handle) = app_handle {
                    use tauri::{Emitter, Manager};
                    if let Some(db) = handle.try_state::<crate::traffic::TrafficDb>() {
                        if let Err(e) = db.update_streaming_log(row_id, &token_data, Some(ttfb), Some(duration_ms)) {
                            log::warn!("流式日志 UPDATE 失败: {}", e);
                        } else {
                            // emit type="update"
                            let mut updated_entry = entry_for_update;
                            updated_entry.input_tokens = token_data.input_tokens;
                            updated_entry.output_tokens = token_data.output_tokens;
                            updated_entry.cache_creation_tokens = token_data.cache_creation_tokens;
                            updated_entry.cache_read_tokens = token_data.cache_read_tokens;
                            updated_entry.stop_reason = token_data.stop_reason;
                            updated_entry.ttfb_ms = Some(ttfb);
                            updated_entry.duration_ms = Some(duration_ms);
                            let payload = crate::traffic::log::TrafficLogPayload::from_entry(row_id, &updated_entry, "update");
                            if let Err(e) = handle.emit("traffic-log", &payload) {
                                log::warn!("emit traffic-log update 失败: {}", e);
                            }
                        }
                    }
                }
            }
            Err(_) => {
                // oneshot sender 被 drop（stream 异常中断，客户端断开）
                log::debug!("流式 token 回传 channel 已关闭（stream 可能被客户端中断）");
            }
        }
    });
}
```
