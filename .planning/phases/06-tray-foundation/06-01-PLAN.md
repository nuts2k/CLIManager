Log warnings on errors using `log::warn!`.

<task type="auto">
  <name>Task 2: Wire tray into lib.rs with close-to-tray lifecycle</name>
  <files>src-tauri/src/lib.rs</files>
  <action>
Refactor `lib.rs` to integrate the tray module and implement close-to-tray lifecycle. This is the critical wiring task.

1. **Add module declaration** at the top with other mods:
   ```rust
   mod tray;
   ```

2. **Add `on_window_event` handler** to the Builder chain, BEFORE `.setup()`:
   ```rust
   .on_window_event(|window, event| {
       if let tauri::WindowEvent::CloseRequested { api, .. } = event {
           api.prevent_close();
           let _ = window.hide();
           #[cfg(target_os = "macos")]
           tray::apply_tray_policy(window.app_handle(), false);
       }
   })
   ```
   This intercepts the close button (red X) and hides the window instead of closing it, then switches to Accessory mode (no Dock, no Cmd+Tab). Per CONTEXT.md: silent, no notification or dialog.

3. **Add tray setup inside `.setup()` closure**, after the existing file watcher setup:
   ```rust
   // Tray setup
   let menu = tray::create_tray_menu(app.handle())
       .map_err(|e| e.to_string())?;

   let icon_bytes: &[u8] = include_bytes!("../icons/tray/tray-icon-template.png");
   let icon = tauri::image::Image::from_bytes(icon_bytes)
       .map_err(|e| e.to_string())?;

   use tauri::tray::{MouseButton, TrayIconBuilder, TrayIconEvent};

   let _tray = TrayIconBuilder::with_id("main")
       .icon(icon)
       .icon_as_template(true)
       .menu(&menu)
       .show_menu_on_left_click(true)
       .on_tray_icon_event(|tray_icon, event| {
           if let TrayIconEvent::DoubleClick { button: MouseButton::Left, .. } = event {
               tray::show_main_window(tray_icon.app_handle());
           }
       })
       .on_menu_event(|app, event| {
           tray::handle_tray_menu_event(app, &event.id.0);
       })
       .build(app)?;
   ```
   Key details:
   - `icon_as_template(true)` enables macOS auto dark/light mode adaptation (TRAY-01)
   - `show_menu_on_left_click(true)` makes single click open menu (macOS standard per CONTEXT.md)
   - DoubleClick handler calls `show_main_window` per CONTEXT.md decision
   - `_tray` binding keeps the tray icon alive for the app lifetime

4. **Refactor from `.run()` to `.build()+.run()`** to enable Cmd+Q distinction:
   Replace:
   ```rust
   .run(tauri::generate_context!())
   .expect("error while running tauri application");
   ```
   With:
   ```rust
   .build(tauri::generate_context!())
   .expect("error while building tauri application");

   app.run(|_app_handle, _event| {});
   ```
   The `.run()` callback receives `RunEvent` including `ExitRequested` (fires on Cmd+Q). By NOT calling `api.prevent_exit()`, Cmd+Q proceeds to quit. The `on_window_event` handler only intercepts close button via `CloseRequested`. This achieves the Cmd+Q vs close button distinction per CONTEXT.md locked decision.

   Note: The `app` variable must be bound with `let app = tauri::Builder::default()...build(...)...;` since we now call `app.run()` separately.

5. **Preserve all existing functionality**: All existing `.plugin()`, `.manage()`, `.invoke_handler()`, and `.setup()` (file watcher) calls remain unchanged. The tray setup is ADDED to `.setup()`, not replacing it. The `.on_window_event()` is new.

   If DoubleClick events conflict with `show_menu_on_left_click(true)` at runtime (Pitfall 4 from research), this is acceptable -- the "打开主窗口" menu item provides the same functionality. Do NOT remove `show_menu_on_left_click(true)` preemptively.
