**AppShell.tsx — 页面视图切换淡入淡出:**

用纯 CSS transition 实现，不引入额外动画库（用户决策留给 Claude 自行决定实现方式）。

方案：使用 CSS `transition: opacity 150ms ease-out` + 条件 className 实现淡入淡出。由于 React 条件渲染（`{view === "main" && ...}`）会直接挂载/卸载组件，需要改为始终渲染两个视图但用 opacity + pointer-events 控制可见性：

将 `<main>` 内部改为：
```tsx
<main className="relative flex-1 overflow-hidden">
  <div
    className={`absolute inset-0 transition-opacity duration-150 ease-out ${
      view === "main" ? "opacity-100" : "opacity-0 pointer-events-none"
    }`}
  >
    <ProviderTabs refreshTrigger={syncKey} />
  </div>
  <div
    className={`absolute inset-0 transition-opacity duration-150 ease-out ${
      view === "settings" ? "opacity-100" : "opacity-0 pointer-events-none"
    }`}
  >
    <SettingsPage onBack={() => setView("main")} onShowImport={handleShowImport} />
  </div>
</main>
```

注意事项：
- `absolute inset-0` 让两个视图叠放在同一位置
- `pointer-events-none` 防止隐藏视图拦截点击
- `duration-150` 约 150ms 的淡入淡出（用户决策指定 ~150ms）
- `ease-out` 缓动函数（Claude 自行选择，ease-out 更自然）
- ProviderTabs 始终渲染（保持其状态不丢失），SettingsPage 也始终渲染（Tab 选择状态保持）
