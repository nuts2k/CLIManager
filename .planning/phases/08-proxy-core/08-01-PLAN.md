    - proxy_handler：非 "PROXY_MANAGED" 的认证头原样保留不修改
    - proxy_handler：请求路径 + query 原样拼接到 upstream base_url 后面
    - proxy_handler：hop-by-hop headers (host, content-length, transfer-encoding) 不转发给上游
    - proxy_handler：上游连接失败时返回 502 + JSON 错误
    - proxy_handler：上游返回的非 2xx 响应原样透传（状态码 + body）
    - proxy_handler：响应使用 bytes_stream() + Body::from_stream() 实现流式透传
  </behavior>
  <action>
    1. 创建 src-tauri/src/proxy/handler.rs：

    2. health_handler：
       - async fn health_handler() -> (StatusCode, Json<Value>)
       - 返回 (200, {"status": "ok"})

    3. proxy_handler 核心转发逻辑：
       - 签名：async fn proxy_handler(State(state): State<ProxyState>, req: axum::extract::Request) -> Result<Response, ProxyError>
       - 步骤 A：从 state.get_upstream() 获取当前上游目标，None 时返回 ProxyError::NoUpstreamConfigured
       - 步骤 B：提取原始请求的 method, path, query, headers
       - 步骤 C：读取请求 body — 使用 axum::body::to_bytes(body, 200 * 1024 * 1024).await（200MB 上限，与 cc-switch 一致）。to_bytes 返回 axum::Error，需要将其转换为 ProxyError::Internal
       - 步骤 D：拼接上游 URL — `format!("{}{}{}", upstream.base_url.trim_end_matches('/'), path, query_string)`。注意 base_url 末尾可能有 `/`，需 trim
       - 步骤 E：构建 reqwest 请求，透传 headers，但跳过 hop-by-hop headers (host, content-length, transfer-encoding, connection)
       - 步骤 F：凭据替换逻辑——遍历 headers 时，如果遇到 x-api-key 或 authorization：
         - 值为 "PROXY_MANAGED" 或 "Bearer PROXY_MANAGED" → 跳过（不转发），后面注入真实凭据
         - 其他值 → 原样转发（非代理管理的请求不修改）
       - 步骤 G：根据 upstream.protocol_type 注入真实凭据：
         - Anthropic → header("x-api-key", &upstream.api_key)
         - OpenAiCompatible → header("Authorization", format!("Bearer {}", upstream.api_key))
       - 步骤 H：发送请求 .body(body_bytes.to_vec()).send().await，错误映射为 ProxyError::UpstreamUnreachable
       - 步骤 I：构建响应——透传上游 status + headers（过滤 transfer-encoding, content-length, connection）
       - 步骤 J：响应体流式透传——Body::from_stream(upstream_resp.bytes_stream())，不缓冲
       - 重要：不加 CompressionLayer，不设 Content-Length header，保留上游的 Content-Type（如 text/event-stream）

    4. 辅助函数 is_hop_by_hop(header_name: &str) -> bool：
       - 匹配 "host", "content-length", "transfer-encoding", "connection"

    5. 更新 proxy/mod.rs：
       - 添加 pub mod handler;
       - 添加 pub use handler::{proxy_handler, health_handler};

    6. #[cfg(test)] 模块：
       - 测试凭据替换逻辑：创建 mock headers 验证替换行为（不需要真实网络请求）
       - 测试 is_hop_by_hop 函数
       - 测试 health_handler 返回值
       - 注意：端到端转发测试在 Task 3 中通过启动真实服务器完成

  <action>
    1. 创建 src-tauri/src/proxy/server.rs：

    2. build_router(state: ProxyState) -> Router 函数：
       - Router::new().route("/health", get(health_handler)).fallback(proxy_handler).with_state(state)

    3. ProxyServer struct：
       - 字段：shutdown_tx: Option<oneshot::Sender<()>>, server_handle: Option<JoinHandle<()>>, state: ProxyState, port: u16
       - new(port: u16, client: reqwest::Client) -> Self：创建新实例，state 为空 upstream
       - state(&self) -> &ProxyState：获取状态引用（用于 update_upstream）

    4. ProxyServer::start(&mut self) -> Result<(), ProxyError>：
       - 检查 shutdown_tx 是否已有值 → AlreadyRunning
       - 创建 oneshot channel
       - TcpListener::bind(format!("127.0.0.1:{}", self.port))，错误映射为 BindFailed
       - 构建 router via build_router(self.state.clone())
       - tokio::spawn axum::serve(listener, app).with_graceful_shutdown(async { rx.await.ok(); })
       - 保存 shutdown_tx 和 server_handle
       - 调用 health_check(self.port).await，失败则自动 stop 并返回错误

    5. ProxyServer::stop(&mut self) -> Result<(), ProxyError>：
       - 取出 shutdown_tx，None 则返回 NotRunning
       - 发送停机信号
       - tokio::time::timeout(5s, handle.await)，超时返回 StopTimeout

    6. ProxyServer::is_running(&self) -> bool：检查 shutdown_tx 是否存在

    7. health_check(port: u16) 异步函数：
       - 创建临时 reqwest::Client
       - GET http://127.0.0.1:{port}/health，超时 3 秒
       - 检查 status == 200，否则返回 HealthCheckFailed

       - test_proxy_forward：启动一个 mock 上游（axum 服务器返回固定 JSON），配置 ProxyState 指向它，通过代理发请求验证转发正确
       - test_credential_replacement_e2e：mock 上游检查收到的 headers 是否包含真实 api_key（而非 PROXY_MANAGED）
       - test_sse_streaming：mock 上游返回 SSE 格式的 chunked 响应（Content-Type: text/event-stream），通过代理接收验证逐 chunk 到达
       - test_upstream_unreachable：配置 upstream 指向不存在的地址，验证返回 502 + JSON 格式错误
       - test_bind_failed：绑定已占用端口，验证返回 BindFailed 错误
       - test_double_start：连续调用两次 start()，验证第二次返回 AlreadyRunning
