<task type="auto" tdd="true">
  <name>Task 1: 实现 ProxyService 多端口管理器</name>
  <files>src-tauri/src/proxy/mod.rs</files>
  <behavior>
    - ProxyService::new() 创建空的管理器，共享一个 reqwest::Client
    - ProxyService::start("claude", 15800, upstream) 启动 Claude Code 代理
    - ProxyService::start("codex", 15801, upstream) 启动 Codex 代理
    - 两个代理可同时运行在不同端口
    - ProxyService::stop("claude") 只停止 Claude Code 代理，不影响 Codex
    - ProxyService::status() 返回所有代理的运行状态
    - ProxyService::update_upstream("claude", new_target) 动态切换上游
    - ProxyService::stop_all() 停止所有正在运行的代理
    - 启动已运行的 cli_id 返回 AlreadyRunning
    - 停止未运行的 cli_id 返回 NotRunning
  </behavior>
  <action>
    1. 在现有 proxy/mod.rs 中扩展（保留已有的子模块声明和 pub use）：

    2. ProxyService struct：
       - servers: tokio::sync::Mutex<HashMap<String, ProxyServer>>（cli_id -> ProxyServer）
       - http_client: reqwest::Client（共享，tcp_keepalive 30s, connect_timeout 10s，不设全局 timeout）

    3. ProxyService::new() -> Self：
       - 创建共享 reqwest::Client（builder 配置 tcp_keepalive + connect_timeout）
       - 空 HashMap

    4. ProxyService::start(&self, cli_id: &str, port: u16, upstream: UpstreamTarget) -> Result<(), ProxyError>：
       - 锁定 servers
       - 检查 cli_id 是否已存在且 is_running() → AlreadyRunning
       - 创建 ProxyServer::new(port, self.http_client.clone())
       - 设置 upstream：server.state().update_upstream(upstream).await
       - 调用 server.start().await
       - 插入 HashMap
       - log::info! 记录启动

    5. ProxyService::stop(&self, cli_id: &str) -> Result<(), ProxyError>：
       - 锁定 servers
       - 获取 server，不存在返回 NotRunning
       - 调用 server.stop().await
       - 从 HashMap 中移除
       - log::info! 记录停止

    6. ProxyService::stop_all(&self) -> Vec<(String, Result<(), ProxyError>)>：
       - 遍历所有 server 逐个 stop
       - 返回每个 cli_id 的停止结果
       - 清空 HashMap

    7. ProxyService::status(&self) -> ProxyStatusInfo：
       - ProxyStatusInfo struct（Serialize）：servers: Vec<ServerStatus>
       - ServerStatus struct（Serialize）：cli_id: String, port: u16, running: bool
       - 遍历 HashMap 收集状态

    8. ProxyService::update_upstream(&self, cli_id: &str, upstream: UpstreamTarget) -> Result<(), ProxyError>：
       - 锁定 servers
       - 获取 server，不存在返回 NotRunning
       - 调用 server.state().update_upstream(upstream).await
       - log::info! 记录上游更新

    9. 公开导出 ProxyService, ProxyStatusInfo, ServerStatus

    10. #[cfg(test)] 模块：
        - test_proxy_service_start_stop：启动两个代理（动态端口），验证 status 显示两个 running，停止一个后另一个仍在
        - test_proxy_service_dual_port：同时启动 claude + codex 在不同端口，各自转发正确
        - test_proxy_service_stop_all：启动多个后 stop_all，验证全部停止
        - test_proxy_service_update_upstream：启动后更新上游目标，验证新请求使用新上游
        - 注意：测试中使用动态端口（127.0.0.1:0），不使用固定的 15800/15801
  </action>
