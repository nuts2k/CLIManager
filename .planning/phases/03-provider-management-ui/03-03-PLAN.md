1. Create `src/hooks/useProviders.ts`:
   - `useProviders(cliId: string)` hook that wraps Tauri provider CRUD calls
   - State: `providers: Provider[]`, `loading: boolean`
   - `refresh()` -- calls `listProviders(cliId)` and updates state
   - `switchProvider(providerId: string)` -- calls `setActiveProvider(cliId, providerId)`, refreshes settings, shows success/error Toast
   - `removeProvider(id: string)` -- calls `deleteProvider(id)`, refreshes providers + settings, shows Toast
   - `copyProvider(provider: Provider)` -- creates a new provider with same config, name + " (copy)", calls `createProvider`, refreshes, shows Toast
   - `copyProviderTo(provider: Provider, targetCliId: string)` -- creates provider in target CLI with name + ` (copy from ${cliId tab name})`, shows Toast
   - `testProviderConnection(providerId: string)` -- calls `testProvider(providerId)`, shows Toast with result including elapsed_ms
   - Auto-refresh on cliId change via useEffect
   - All operations use `useTranslation()` for Toast messages

2. Create `src/hooks/useSettings.ts`:
   - `useSettings()` hook that loads and manages LocalSettings
   - State: `settings: LocalSettings | null`, `loading: boolean`
   - `refresh()` -- calls `getLocalSettings()`
   - `updateSettings(partial)` -- merges partial changes, calls `updateLocalSettings`
   - `getActiveProviderId(cliId: string)` -- returns `settings?.active_providers[cliId] ?? null`
   - Auto-load on mount

   - When view === "settings", renders SettingsPage (placeholder div for now, built in Plan 04)
