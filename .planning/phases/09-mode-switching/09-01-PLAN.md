新增内部函数 `_proxy_enable_in` 和 `_proxy_disable_in`（接收 path 参数用于测试）：

**_proxy_enable_in(providers_dir, local_settings_path, cli_id, proxy_service, adapter):**
- 读取 local.json 获取 active_providers[cli_id]
- 如果无活跃 Provider，返回错误 "该 CLI 无活跃 Provider，无法开启代理"
- 从 iCloud 读取 Provider 获取真实凭据
- 获取端口 proxy_port_for_cli(cli_id)，无则返回错误
- 构造代理专用 Provider（临时构造，不保存到 iCloud）：
  - Claude: api_key="PROXY_MANAGED", base_url="http://127.0.0.1:15800", protocol_type=Anthropic
  - Codex: api_key="PROXY_MANAGED", base_url="http://127.0.0.1:15801", protocol_type=OpenAiCompatible
- 调用 adapter.patch(proxy_provider) 将 CLI 配置 patch 为代理凭据
- 构造 UpstreamTarget（真实 api_key + base_url + protocol_type）
- 调用 proxy_service.start(cli_id, port, upstream)
- **如果 start() 失败**：调用 adapter.patch(real_provider) 回滚 CLI 配置，返回错误
- 更新 local.json：proxy_takeover.cli_ids 添加 cli_id，proxy.cli_enabled[cli_id]=true
- 写回 local.json

**_proxy_disable_in(providers_dir, local_settings_path, cli_id, proxy_service, adapter):**
- 读取 local.json 获取 active_providers[cli_id]
- 从 iCloud 读取当前活跃 Provider
- 调用 adapter.patch(real_provider) 还原 CLI 配置为真实凭据
  - 如果 patch 失败，尝试 adapter.clear()，记录日志
- 调用 proxy_service.stop(cli_id)（best-effort，不影响配置还原结果）
- 更新 local.json：proxy_takeover.cli_ids 移除 cli_id，proxy.cli_enabled[cli_id]=false
- 写回 local.json

**proxy_enable Tauri 命令（async）：**
- 注入 `State<ProxyService>`
- 读取真实 providers_dir 和 local_settings_path
- 调用 _proxy_enable_in（adapter 传 None 使用真实 adapter）
- 发射 `proxy-mode-changed` 事件（使用 AppHandle）

**proxy_disable Tauri 命令（async）：**
- 类似 proxy_enable，调用 _proxy_disable_in

**proxy_set_global Tauri 命令（async）：**
- 注入 `State<ProxyService>`
- 读取 local.json
- 更新 proxy.global_enabled
- 如果 enabled=true：对 proxy.cli_enabled 中所有 true 的 CLI 调用 _proxy_enable_in（跳过已运行的）
- 如果 enabled=false：对 proxy_takeover.cli_ids 中所有 CLI 调用 _proxy_disable_in
- 写回 local.json
- 发射 `proxy-mode-changed` 事件

**proxy_get_mode_status Tauri 命令（async）：**
- 注入 `State<ProxyService>`
- 读取 local.json 获取 proxy settings
- 获取 proxy_service.status() 获取实际运行状态
- 组合返回 ProxyModeStatus
