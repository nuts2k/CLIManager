1. Add `pub mod codex;` to `src-tauri/src/adapter/mod.rs`.

2. Also add a `restore_from_backup` utility function to `src-tauri/src/adapter/mod.rs`:
   - `pub fn restore_from_backup(target: &Path, backup_dir: &Path) -> Result<(), AppError>`
   - Lists .bak files in backup_dir that start with the target filename
   - Sorts descending (newest first), takes the first
   - `fs::copy` from backup to target
   - If no backup found, return `AppError::Validation("No backup found for rollback")`

3. Implement `src-tauri/src/adapter/codex.rs`:

   `CodexAdapter` struct:
   - `config_dir: PathBuf` -- directory containing auth.json and config.toml (default: `~/.codex/`)
   - `backup_dir: PathBuf` -- backup destination (default: `~/.cli-manager/backups/codex/`)

   Constructor pattern:
   - `pub fn new() -> Self` -- uses default dirs
   - `pub fn new_with_paths(config_dir: PathBuf, backup_dir: PathBuf) -> Self` -- for testing

   `impl CliAdapter for CodexAdapter`:
   - `cli_name()` returns `"codex"`
   - `patch()` method implementing two-phase write:

     a. Compute paths: `auth_path = config_dir.join("auth.json")`, `config_path = config_dir.join("config.toml")`

     b. Read existing content for both files (or defaults: `"{}"` for auth.json, `""` for config.toml)

     c. **Pre-validation:** Parse auth.json as `serde_json::Value`, parse config.toml as `toml_edit::DocumentMut`. Return error on failure.

     d. **Backup both files** (if they exist) before any writes. Call `create_backup` + `rotate_backups(5)` for each.

     e. **Surgical patch auth.json:** `patch_codex_auth_json(content, api_key)` -- Value-level merge, only sets `OPENAI_API_KEY` at top level

     f. **Post-validate auth.json:** Parse patched string as Value

     g. **Phase 1 write:** `atomic_write(&auth_path, auth_patched.as_bytes())`

     h. **Surgical patch config.toml:** `patch_codex_toml(content, base_url)` -- use `toml_edit::DocumentMut`, set `doc["base_url"] = toml_edit::value(base_url)`

     i. **Post-validate config.toml:** Parse patched string as DocumentMut

     j. **Phase 2 write:** `atomic_write(&config_path, toml_patched.as_bytes())`
        - If Phase 2 fails: call `restore_from_backup(&auth_path, &backup_dir)` then return the original error

     k. Return `PatchResult` with both files

   Internal functions:
   - `patch_codex_auth_json(existing: &str, api_key: &str) -> Result<String, AppError>` -- parse as Value, insert OPENAI_API_KEY, serialize pretty
   - `patch_codex_toml(existing: &str, base_url: &str) -> Result<String, AppError>` -- parse as DocumentMut, set base_url, doc.to_string()

4. Tests use `tempfile::TempDir`. For TOML comment preservation test, use a config.toml like:

   ```toml
   base_url = "https://old.example.com"

   [projects]

   ```
   After patching, assert the output still contains `# This is a user comment` and `# Project settings`.

5. For rollback test: write auth.json, then make config.toml path point to a directory (so atomic_write fails), verify auth.json content matches the pre-patch backup.
