Implement `src-tauri/src/adapter/claude.rs`:

1. `ClaudeAdapter` struct with fields:
   - `config_dir: PathBuf` -- directory containing settings.json (default: `~/.claude/`)
   - `backup_dir: PathBuf` -- backup destination (default: `~/.cli-manager/backups/claude/`)

2. Constructor pattern following `_in/_to` convention:
   - `pub fn new() -> Self` -- uses default dirs (home_dir based)
   - `pub fn new_with_paths(config_dir: PathBuf, backup_dir: PathBuf) -> Self` -- for testing

3. `impl CliAdapter for ClaudeAdapter`:
   - `cli_name()` returns `"claude"`
   - `patch()` method implementing the full flow:
     a. Compute `settings_path = config_dir.join("settings.json")`
     b. Read existing content or use `"{}"` if file does not exist
     c. **Pre-validation:** `serde_json::from_str::<Value>(&content)` -- return `AppError::Validation` on failure
     d. **Backup:** If file exists, call `create_backup(&settings_path, &backup_dir)`, then `rotate_backups(&backup_dir, 5)`
     e. **Surgical patch:** `patch_claude_json(&content, &provider.api_key, &provider.base_url)`
     f. **Post-validation:** Parse the patched string as `Value` to confirm valid JSON
     g. **Write:** `atomic_write(&settings_path, patched.as_bytes())`
     h. Return `PatchResult`

4. Internal `patch_claude_json(existing: &str, api_key: &str, base_url: &str) -> Result<String, AppError>`:
   - Parse as `serde_json::Value`
   - Navigate to root object, ensure `"env"` key exists (create Object if missing)
   - Insert `"ANTHROPIC_AUTH_TOKEN"` and `"ANTHROPIC_BASE_URL"` into env object
   - Serialize back with `serde_json::to_string_pretty`
   - This preserves all other keys at any nesting level

5. All tests use `tempfile::TempDir` for isolation. Write sample settings.json files with known content (extra keys, nested objects, specific key ordering) and assert they survive patching.
