    **2. 创建 TrafficTrendChart.tsx：**

    使用 recharts 2.x API 实现 ComposedChart 双轴图（RESEARCH.md Pattern 4）。

    props: `{ data: TimeStat[]; timeRange: TimeRange }`

    组件实现要点：
    - ResponsiveContainer width="100%" height={240}
    - ComposedChart，margin 适当留白
    - CartesianGrid strokeDasharray="3 3"，stroke 使用 var(--color-border)
    - XAxis dataKey="label"，tick fill 使用 var(--color-muted-foreground)
    - 左 YAxis (yAxisId="requests")：请求数，orientation="left"
    - 右 YAxis (yAxisId="tokens")：Token 总量，orientation="right"
    - Bar (yAxisId="requests", dataKey="request_count")：fill 使用 var(--color-chart-1)，radius=[2,2,0,0]
    - Line (yAxisId="tokens", dataKey="total_tokens")：stroke 使用 var(--color-chart-2)，dot=false，strokeWidth=2
    - Tooltip：contentStyle 使用 CSS 变量（background: var(--color-popover)，border: var(--color-border)，color: var(--color-foreground)）
    - Legend：wrapperStyle 使用 var(--color-muted-foreground)

    **数据点填充逻辑（在组件内或 hook 内实现）：**

    RESEARCH.md Pitfall 5 指出后端 GROUP BY 不产生空行，前端需填充缺失时间点。

    24h 模式：生成 24 个小时标签（"00:00" ~ "23:00"），用后端数据覆盖对应项，缺失的填 request_count=0, total_tokens=0。

    7d 模式：生成 7 个天标签（从今天往前 6 天），格式 "MM-DD"（label 需对齐后端返回的 "YYYY-MM-DD" 格式，用 slice(5) 缩短显示），缺失的填 0。

    将此填充逻辑实现为 `buildHourlyData(raw: TimeStat[]): TimeStat[]` 和 `buildDailyData(raw: TimeStat[]): TimeStat[]` 纯函数，放在 TrafficTrendChart.tsx 文件内或 formatters.ts 中。

    空数据处理：如果所有点的 request_count 和 total_tokens 均为 0，图表仍然渲染（X 轴连续，Y 轴显示 0），不需要特殊空状态。

    - en: "analysis.chartRequests": "Requests", "analysis.chartTokens": "Total Tokens"
    或者直接在 Bar/Line 的 name prop 中使用 t() 函数。
