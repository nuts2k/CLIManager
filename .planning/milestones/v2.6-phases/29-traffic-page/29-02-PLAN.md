1. **创建 `src/components/traffic/formatters.ts`**（纯函数工具，无 React 依赖）：
   - `formatTokenCount(n: number | null): string` — null 返回 "--"，>= 1_000_000 返回 "X.XM"，>= 1_000 返回 "X.Xk"，否则返回原始数字字符串
   - `formatTime(epochMs: number): string` — diff < 60s 返回 "{N}秒前"（用 i18n t 函数不合适，此处返回纯数字，i18n 在组件层做），diff < 3600s 返回分钟数，否则返回 "HH:MM:SS"。**但考虑 i18n：** 此函数返回 `{ type: "seconds", count: N } | { type: "minutes", count: N } | { type: "absolute", value: "HH:MM:SS" }`，让组件层用 t() 拼接。
   - `formatDuration(ms: number | null): string` — null 返回 "--"，>= 1000 返回 "X.Xs"，否则返回 "Xms"
   - `calcTps(outputTokens: number | null, durationMs: number | null): string` — 任一为 null 或 durationMs === 0 返回 "--"，否则返回 "XX.X"（不含 "t/s" 后缀，后缀在组件层通过 i18n 拼接）
   - `statusCodeClass(code: number | null): string` — null 返回 "text-muted-foreground"，2xx 返回 "text-status-success"，4xx 返回 "text-status-warning"，5xx 返回 "text-destructive"，其他返回 "text-foreground"

   - Props：`{ logs: TrafficLog[] }`（已经是筛选后的日志列表）
   - 使用 `useMemo` 计算 5 个指标（基于 24 小时窗口过滤：`Date.now() - 24 * 60 * 60 * 1000`）：
     - `total`：窗口内总数
     - `successRate`：status_code 200-299 的比例（total > 0 时计算，否则 0）
     - `inputTokens`：sum(input_tokens ?? 0)
     - `outputTokens`：sum(output_tokens ?? 0)
     - `cacheHitRate`：有 cache_read_tokens > 0 的条目占比
   - 5 张卡片横排布局：使用 shadcn Card 组件，`grid grid-cols-5 gap-3`
   - 每张卡片布局：图标（lucide，size-4，text-muted-foreground）+ 标签（text-xs text-muted-foreground）+ 大号数值（text-2xl font-bold）+ 微小趋势线
   - 图标选择：请求数 → `Activity`，Input Token → `ArrowDownRight`，Output Token → `ArrowUpRight`，成功率 → `Shield`（或 `CheckCircle`），缓存命中率 → `Database`（或 `Zap`）
   - 趋势线实现：轻量 inline SVG sparkline。将窗口内日志按时间分为 12 个桶（每桶 2 小时），统计每桶的请求数，绘制 polyline。SVG 宽约 60px，高 20px，stroke 使用 `currentColor` + `opacity-30`。如果数据不足 2 个桶则不显示趋势线。
   - 成功率和缓存命中率显示为百分比（如 "98.5%"），token 使用 formatTokenCount 格式化
   - 卡片底部小字标注 `t("traffic.statsBasis")`（"基于当前缓存"）
   - 所有文字使用 `useTranslation()` 的 `t()` 函数
