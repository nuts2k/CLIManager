From src-tauri/src/lib.rs (line 94):
```rust
let icon_bytes: &[u8] = include_bytes!("../icons/tray/tray-icon-template.png");

```

注意：lib.rs 只引用 tray-icon-template.png（@1x），@2x 版本由 macOS 自动在 Retina 屏幕上选取（基于文件名约定）。但由于 include_bytes! 是编译时嵌入，@2x 不会自动加载。@2x 文件作为预备放置，若后续改为运行时加载可用。当前 lib.rs 只使用 @1x 文件，因此 @1x 版本必须在 44x44 大小下清晰（当前现有文件就是 44x44）。

实际情况：现有 tray-icon-template.png 为 44x44（即 @2x Retina 尺寸），include_bytes! 直接嵌入 44x44 图标，macOS 菜单栏渲染时缩放到 22pt 显示。因此新图标也应保持 44x44 尺寸作为主文件。

    **步骤 2 — 生成托盘图标 SVG：**
    用 Write 工具创建 src-tauri/icons/tray/tray-icon.svg，设计要求：
    - 画布 44x44（匹配 @2x Retina 尺寸，也是现有文件尺寸）
    - 透明背景（template 图标必须是透明背景 + 黑色图案）
    - 保留应用图标的核心结构：中心枢纽圆 + 外围小节点 + 连接线
    - 所有图案元素使用纯黑色 #000000（macOS template 模式会自动根据菜单栏深浅反转颜色）
    - 简化设计：减少节点数量（3-4 个即可），线条适当加粗确保 22px 显示时仍清晰
    - 中心圆大小约 8-10px（在 44px 画布上），外围节点约 3-4px
    - 连接线宽度约 2-3px
    - 节点位置与应用图标保持相似的不均匀有机分布
    - 关键：在 22px 缩放后整体图案必须清晰可辨，避免过细线条和过小元素

    **步骤 3 — 转换为 PNG：**
    将 SVG 转换为透明背景 PNG：
    ```bash
    # @2x 版本（44x44，也是 include_bytes! 实际使用的文件）
    qlmanage -t -s 44 -o /tmp/ src-tauri/icons/tray/tray-icon.svg
    cp /tmp/tray-icon.svg.png src-tauri/icons/tray/tray-icon-template@2x.png

    # @1x 版本（22x22）
    sips -z 22 22 src-tauri/icons/tray/tray-icon-template@2x.png --out src-tauri/icons/tray/tray-icon-template.png
    ```

    注意：由于现有 tray-icon-template.png 实际为 44x44，而 lib.rs include_bytes! 嵌入的就是这个文件，新的 @1x 文件也应保持 44x44 以确保兼容性。如果验证发现 qlmanage 无法正确渲染透明背景 SVG，使用 Python cairosvg 或直接用 Python Pillow 从 SVG 坐标手工绘制 PNG 作为备选方案。

    实际尺寸策略：
    - tray-icon-template.png: 保持 44x44（与现有文件一致，lib.rs include_bytes! 直接使用）
    - tray-icon-template@2x.png: 44x44（同尺寸备份，供未来高 DPI 使用）

    **步骤 4 — 验证 template 属性：**

    - 透明背景（alpha 通道）
    - 图案部分使用黑色或深灰色
    - 无彩色元素
    - 文件大小合理（几百字节到几 KB）

    **步骤 5 — 清理：**
    保留 tray-icon.svg 源文件作参考。清理 /tmp/ 临时文件。
