<task type="auto">
  <name>Task 1: watcher process_events 新增代理模式感知（LIVE-02）</name>
  <files>src-tauri/src/watcher/mod.rs</files>
  <action>
在 `process_events` 函数中，`sync_changed_active_providers` 调用之后、构造 payload 之前，新增代理上游更新逻辑：

1. 从 `app_handle` 获取 `ProxyService` 状态：`app_handle.state::<crate::proxy::ProxyService>()`
2. 读取 local.json 获取 `proxy_takeover`、`active_providers` 和 `proxy.global_enabled`
3. 对于每个处于代理模式的 CLI（`proxy_takeover.cli_ids` 中的 cli_id）：
   - 获取该 CLI 的活跃 Provider ID（从 `active_providers`）
   - 检查该 Provider ID 是否在 `changed_files` 中（changed_files 是文件 stem，即 provider id）
   - 如果是，从 iCloud 重新读取该 Provider 文件，构造 `UpstreamTarget`
   - 使用 `tauri::async_runtime::spawn` 提交 `proxy_service.update_upstream(cli_id, upstream)` 异步操作

**关键约束：**
- `process_events` 是 notify 回调中的同步函数，不能直接 `.await`
- 使用 `tauri::async_runtime::spawn` 提交 async 任务（与 Phase 9 的 `restore_proxy_state` 模式一致）
- `proxy_service` 需要 clone（ProxyService 内部是 Arc<Mutex<...>>，支持 Clone）
- 即使 update_upstream 失败也只 log，不影响正常的事件处理流程
- 注意 `changed_files` 的元素是文件 stem（即 provider id），直接和 `active_providers` 的 value 比较

**代码模式参考（来自 RESEARCH.md）：**
```rust
let proxy_service = app_handle.state::<crate::proxy::ProxyService>().inner().clone();
tauri::async_runtime::spawn(async move {
    proxy_service.update_upstream(&cli_id, upstream).await.ok();
});
```

注意：需要在 watcher/mod.rs 添加 `use tauri::Manager;`（如果尚未导入），因为 `app_handle.state()` 需要 Manager trait。检查文件头部 — 已有 `use tauri::{AppHandle, Emitter, Manager};`，Manager 已导入。

   - 如果匹配，从更新后的 provider 构造 `UpstreamTarget { api_key, base_url, protocol_type }`
   - 调用 `proxy_service.update_upstream(&cli_id, upstream).await`，失败只 log

**2b. delete_provider 改造：**

1. 将 `delete_provider` 从同步命令改为 async 命令：
   - 签名变为 `pub async fn delete_provider(app_handle: tauri::AppHandle, id: String, proxy_service: State<'_, crate::proxy::ProxyService>) -> Result<(), AppError>`

   - 读取 local.json 的 `proxy_takeover.cli_ids` 和 `active_providers`

   - 如果匹配：先调用 `crate::commands::proxy::_proxy_disable_in(providers_dir, settings_path, &cli_id, &proxy_service, None).await?` 关闭代理
   - 然后 emit `proxy-mode-changed` 事件（需要 `app_handle.emit("proxy-mode-changed", ()).ok();`，需要 `use tauri::Emitter;` — 检查文件头部导入）
