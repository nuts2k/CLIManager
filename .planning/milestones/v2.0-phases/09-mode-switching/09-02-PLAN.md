1. **commands/proxy.rs** — 新增两个公共函数（非 Tauri 命令，供 lib.rs 调用）：

**cleanup_on_exit(providers_dir, local_settings_path, adapter_factory, proxy_service):**
- 读取 local.json，获取 proxy_takeover.cli_ids
- 如果 cli_ids 为空，直接返回（无需清理）
- 对每个 cli_id：
  - 读取 active_providers[cli_id] 获取 provider_id
  - 从 iCloud 读取 Provider
  - 调用 adapter.patch(real_provider) 还原 CLI 配置（adapter 通过 adapter_factory 或 get_adapter_for_cli 获取）
  - 如果 patch 失败，记录 log::error!，继续处理下一个（best-effort）
- 清除 proxy_takeover（设为 None 或空 cli_ids）
- 写回 local.json
- 调用 proxy_service.stop_all()（async，需要通过 tauri::async_runtime::block_on 执行）
- 注意：这个函数需要 **同步** 调用 adapter.patch()（已经是同步的）和 **异步** 调用 proxy_service.stop_all()
- 因为 RunEvent 回调在主线程且不能 `.await`，该函数拆分为：
  - `cleanup_on_exit_sync(providers_dir, local_settings_path)` — 同步部分：读 takeover、adapter.patch 还原、清除 takeover、写 local.json
  - 异步部分（stop_all）在 lib.rs 的回调中通过 `tauri::async_runtime::block_on` 执行

**recover_on_startup(providers_dir, local_settings_path):**
- 同步函数
- 读取 local.json
- 检查 proxy_takeover.cli_ids 是否非空
- 如果非空：对每个 cli_id 执行 adapter.patch(active_provider) 还原
- 清除 proxy_takeover，写回 local.json
- log::info! 记录恢复操作（不弹通知，per CONTEXT.md 决定）
- 返回 Result<(), AppError>

**restore_proxy_state(providers_dir, local_settings_path, proxy_service):**
- async 函数
- 读取 local.json 的 proxy.global_enabled 和 proxy.cli_enabled
- 如果 global_enabled=true：
  - 对每个 cli_enabled[cli_id]=true 的 CLI：
    - 检查是否有活跃 Provider
    - 如果有，调用 _proxy_enable_in 重新开启代理
    - 如果失败，log::warn! 记录，继续下一个
- 返回 Result<(), AppError>

**setup 闭包中（file watcher 之后、tray 创建之前）添加启动时逻辑：**

// 崩溃恢复：检测并还原遗留 takeover
let providers_dir = crate::storage::icloud::get_icloud_providers_dir()?;

注意：ProxyService 当前不实现 Clone（内部 Mutex 不 Clone）。解决方案：
- 方案 A：在 setup 闭包中直接 spawn 一个 async 任务来执行 restore_proxy_state，通过 app.handle() 获取 State
- 方案 B：使用 `tauri::async_runtime::spawn` + `app.handle().clone()` 来执行

推荐方案 A：
```rust
let handle = app.handle().clone();
tauri::async_runtime::spawn(async move {
    let proxy_service = handle.state::<proxy::ProxyService>();
    if let Err(e) = commands::proxy::restore_proxy_state(
        &providers_dir_clone, &local_settings_path_clone, &proxy_service
    ).await {
        log::error!("代理状态恢复失败: {}", e);
    }
});
```

**app.run() 闭包中添加退出事件处理：**
```rust
app.run(|app_handle, event| {
    match event {
        tauri::RunEvent::ExitRequested { .. } => {
            // 正常退出：还原所有 CLI 配置
            let providers_dir = crate::storage::icloud::get_icloud_providers_dir().ok();
            let local_settings_path = crate::storage::local::get_local_settings_path();
            if let Some(providers_dir) = providers_dir {
                commands::proxy::cleanup_on_exit_sync(&providers_dir, &local_settings_path);
            }
            // 异步停止代理
            let proxy_service = app_handle.state::<proxy::ProxyService>();
            tauri::async_runtime::block_on(async {
                proxy_service.stop_all().await;
            });
        }
        _ => {}
    }
});
```

注意关键约束：
- RunEvent::ExitRequested 回调参数是 `&AppHandle`（非 `&mut`），可以读 State
- `tauri::async_runtime::block_on` 可以在同步回调中执行异步代码
- 先还原配置（同步）再停代理（异步）— 确保 CLI 不会在代理停止后仍指向已关闭的 localhost
- 不调用 `api.prevent_exit()` — 允许退出正常进行
