Rewrite `tray.rs` to extend from Phase 6 static menu to Phase 7 dynamic menu. Keep all existing public functions (`show_main_window`, `apply_tray_policy`). Changes:

1. Add imports: `use tauri::menu::CheckMenuItem;` and `use crate::error::AppError;`

2. Add `TrayTexts` struct with `from_language(lang: &str) -> Self`. Fields: `show_main`, `quit`, `claude_header`, `codex_header` -- all `&'static str`. Default to Chinese ("zh"), English for `lang.starts_with("en")`. Brand names "Claude Code" and "Codex" are identical in all languages per CONTEXT.md.

3. Add `parse_provider_event(event_id: &str) -> Option<(&str, &str)>` -- iterate over `[("claude_", "claude"), ("codex_", "codex")]`, use `strip_prefix` to extract provider ID. Return `None` for `_header` suffixes and empty provider IDs. Per CONTEXT.md decision, use `strip_prefix` NOT `split_once`.

4. Rewrite `create_tray_menu(app: &AppHandle) -> Result<Menu<Wry>, Box<dyn std::error::Error>>` to:
   - Read `LocalSettings` via `crate::storage::local::read_local_settings()` (OK to unwrap_or_default on error for graceful degradation)
   - Read providers via `crate::storage::icloud::list_providers()` (default to empty vec on error)
   - Get language from `settings.language.as_deref().unwrap_or("zh")`
   - Build menu in exact order per CONTEXT.md locked decision:
     - "Open Main Window" (or Chinese equivalent) -> separator
     - Claude Code group (if non-empty): disabled MenuItem header -> CheckMenuItems for providers
     - Codex group (if non-empty): disabled MenuItem header -> CheckMenuItems for providers
     - separator (only if any providers existed) -> "Quit"
   - Within each group: sort active provider first, then remaining by `name.to_lowercase()` alphabetically
   - Use `CheckMenuItem::with_id(app, format!("{cli_id}_{provider.id}"), &provider.name, true, is_active, None::<&str>)`
   - Hide empty CLI groups (per locked decision)
   - When all CLIs have zero providers, show only "Open Main Window" and "Quit" (per locked decision)

5. Add `pub fn update_tray_menu(app: &AppHandle)` -- calls `create_tray_menu`, then `app.tray_by_id("main").unwrap().set_menu(Some(menu))`. Log errors, never panic.

6. Add `fn handle_provider_click(app: &AppHandle, cli_id: &str, provider_id: &str)` -- uses `tauri::async_runtime::spawn_blocking` internally. Inside the closure:
   - Get `providers_dir` from `crate::storage::icloud::get_icloud_providers_dir()`
   - Get `settings_path` from `crate::storage::local::get_local_settings_path()`
   - Call `crate::commands::provider::_set_active_provider_in(&providers_dir, &settings_path, cli_id.to_string(), Some(provider_id.to_string()), None)`
   - On success: call `update_tray_menu(&app_handle)` + emit `"providers-changed"` event (reuse existing event name so frontend listeners pick it up without changes -- per research Open Question 1 recommendation)
   - On failure: log error + still call `update_tray_menu` to reset CheckMenuItem visual state (per pitfall 2 in RESEARCH.md)

7. Update `handle_tray_menu_event` to handle new provider click IDs via `parse_provider_event`. Keep existing "show_main" and "quit" handlers unchanged.

8. Add `#[cfg(test)] mod tests` with unit tests for `TrayTexts::from_language` and `parse_provider_event` as specified in behavior block.
