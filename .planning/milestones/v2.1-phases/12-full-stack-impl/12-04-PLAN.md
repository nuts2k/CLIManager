  <files>.claude/commands/ship.md</files>
  <action>
创建 `.claude/commands/` 目录和 `ship.md` 文件。

这是一个 Claude 自定义命令文件，用户通过 `/ship patch`、`/ship minor`、`/ship major` 调用。文件内容是给 Claude 的指令，让 Claude 执行发版流程。

**文件内容要点（给 Claude 的指令）：**

1. **参数解析：** 从 `$ARGUMENTS` 获取 bump 类型（patch/minor/major），默认 patch

2. **版本 bump：**
   - 读取 `src-tauri/Cargo.toml` 中的 `version = "x.y.z"`
   - 根据 bump 类型计算新版本号
   - 用 sed 替换 Cargo.toml 中的版本号
   - 仅修改 Cargo.toml（用户决策：tauri.conf.json 省略 version 字段，无需修改）

3. **CHANGELOG 生成（用户决策：零外部依赖，中文）：**
   - 找到上一个 tag（`git describe --tags --abbrev=0 HEAD` 或如无 tag 则用全部 log）
   - 解析 `git log --pretty=format:"%s" $PREV_TAG..HEAD` 获取提交信息
   - 按 Conventional Commits 规范分类：
     - `feat:` → "新功能"
     - `fix:` → "修复"
     - `refactor:` → "重构"
     - `docs:` → "文档"
     - `chore:` → "其他"
     - 不匹配规范的提交归入"其他"
   - 生成格式化的变更日志文本（中文标题），新版本追加到 CHANGELOG.md 文件头部：
     ```
     ## v{NEW_VERSION} ({YYYY-MM-DD})

     ### 新功能
     - 描述1
     - 描述2

     ### 修复
     - 描述1
     ```
   - 如果 CHANGELOG.md 不存在则创建

4. **Git 操作（用户决策：一键执行，不逐步确认）：**
   - `git add src-tauri/Cargo.toml CHANGELOG.md`
   - `git commit -m "chore(release): v{NEW_VERSION}"`
   - `git tag "v{NEW_VERSION}"`
   - `git push && git push --tags`

5. **输出格式（用户具体要求：checkmark 标记每步状态）：**
   每完成一步输出状态：
   ```
   ✅ 版本 bump: 0.2.0 → 0.3.0
   ✅ CHANGELOG.md 已更新
   ✅ 提交: chore(release): v0.3.0
   ✅ Tag: v0.3.0
   ✅ 已推送到远程仓库
   🚀 发版完成！CI 将自动构建并发布到 GitHub Releases。
   ```

6. **错误处理（Claude 自行决策）：**
   - 如果工作区有未提交的更改，提示用户先提交或 stash
   - 如果 tag 已存在，报错并终止
   - 如果 push 失败，提示用户手动处理（不自动回滚，因为 commit 和 tag 是本地操作可手动删除）

注意：这是 `.md` 文件（Claude 指令），不是 `.sh` 脚本。Claude 读取指令后用 bash 工具执行各步骤。命令中应明确告知 Claude 使用 bash 工具完成所有操作。
