创建 `.github/workflows/` 目录和 `release.yml` 文件，内容实现：

1. **触发条件（CICD-01）：**
   ```yaml
   on:
     push:
       tags:
         - 'v[0-9]*.[0-9]*.[0-9]*'
   ```
   仅匹配三段式 tag（v0.2.0），不匹配 GSD 两段式 tag（v2.1）。

2. **权限和并发控制：**
   ```yaml
   permissions:
     contents: write
   concurrency:
     group: release-${{ github.ref_name }}
     cancel-in-progress: true
   ```

3. **双架构矩阵策略（CICD-02）：**
   ```yaml
   strategy:
     fail-fast: false
     matrix:
       include:
         - platform: 'macos-latest'
           args: '--target aarch64-apple-darwin'
         - platform: 'macos-latest'
           args: '--target x86_64-apple-darwin'
   ```
   使用 `macos-latest`（Apple Silicon runner），`fail-fast: false` 防止一个架构失败导致另一个取消。

4. **步骤（按顺序）：**
   - `actions/checkout@v4`
   - `pnpm/action-setup@v4` with `version: 10`
   - `actions/setup-node@v4` with `node-version: 22, cache: 'pnpm'`
   - `dtolnay/rust-toolchain@stable` with `targets: aarch64-apple-darwin,x86_64-apple-darwin`
   - `pnpm install --frozen-lockfile`
   - `tauri-apps/tauri-action@v1`（CICD-03）

5. **tauri-action 配置（CICD-03、SIGN-01、REL-03）：**
   ```yaml
   - uses: tauri-apps/tauri-action@v1
     env:
       GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
       TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
       APPLE_SIGNING_IDENTITY: "-"
     with:
       tagName: ${{ github.ref_name }}
       releaseName: 'CLIManager ${{ github.ref_name }}'
       releaseDraft: true
       releaseBody: |
         ## CLIManager ${{ github.ref_name }}

         完整更新日志请查看 [CHANGELOG.md](https://github.com/nuts2k/CLIManager/blob/main/CHANGELOG.md)

         <details>
         <summary>macOS 首次安装 Gatekeeper 指引</summary>

         macOS 对未经公证的应用会提示"已损坏"或"无法验证开发者"，请执行以下命令解除限制：

         ```bash
         xattr -cr "/Applications/CLIManager.app"
         ```

         然后在「系统设置 → 隐私与安全性」中点击「仍要打开」。
         </details>
       args: ${{ matrix.args }}
   ```

**注意事项：**
- 不添加 Rust/pnpm 缓存步骤（初版简洁优先，后续可优化）
- `APPLE_SIGNING_IDENTITY: "-"` 表示 ad-hoc 签名（SIGN-01）
- `releaseDraft: true` 让用户可在发布前审查 Release Notes
- releaseBody 包含 Gatekeeper 指引折叠段落（REL-03）
- releaseBody 中链接 CHANGELOG.md 而非内联日志（与 /ship 技能配合）
