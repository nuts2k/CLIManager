  <name>Anthropic → Responses API 请求转换纯函数</name>
  <files>src-tauri/src/proxy/translate/responses_request.rs, src-tauri/src/proxy/translate/mod.rs</files>
  <behavior>
    TDD 行为规格（先写测试，再实现）：

    1. test_basic_text_request: 基础文本请求转换
       - 输入: { model:"claude-3-5-sonnet", max_tokens:1024, system:"You are helpful", messages:[{role:"user",content:[{type:"text",text:"Hello"}]}] }
       - 期望: { model:"claude-3-5-sonnet", max_output_tokens:1024, instructions:"You are helpful", input:[{role:"user",content:"Hello"}] }

    2. test_system_array_format: system 数组格式转 instructions
       - 输入: system:[{type:"text",text:"Part1"},{type:"text",text:"Part2"}]
       - 期望: instructions:"Part1\nPart2"（多段拼接）

    3. test_max_tokens_mapping: max_tokens → max_output_tokens
       - 输入: { max_tokens: 4096 }
       - 期望: result 中有 max_output_tokens:4096，无 max_tokens 字段

    4. test_tools_no_function_wrapper: 工具定义无 function 包装层
       - 输入: tools:[{name:"get_weather", type:"custom", input_schema:{type:"object",properties:{location:{type:"string"}}}}]
       - 期望: tools:[{type:"function", name:"get_weather", parameters:{type:"object",properties:{location:{type:"string"}}}}]（无 function:{} 外包装）

    5. test_tool_result_to_function_call_output: tool_result → function_call_output
       - 输入: messages 包含 {role:"user",content:[{type:"tool_result",tool_use_id:"call_123",content:"result text"}]}
       - 期望: input 包含 {type:"function_call_output", call_id:"call_123", output:"result text"} 作为独立项

    6. test_assistant_tool_use_to_function_call: assistant 工具调用 → function_call
       - 输入: messages 包含 {role:"assistant",content:[{type:"tool_use",id:"call_abc",name:"get_weather",input:{location:"Tokyo"}}]}
       - 期望: input 包含 {type:"function_call", call_id:"call_abc", name:"get_weather", arguments:"{\"location\":\"Tokyo\"}"} 作为独立项

    7. test_multi_turn_conversation: 多轮对话完整转换
       - 输入: user 文本 → assistant 工具调用 → user tool_result → assistant 文本
       - 期望: input 数组包含 user msg → function_call → function_call_output → assistant msg，顺序正确

    8. test_stream_passthrough: stream 标志透传
       - 输入: { stream: true }
       - 期望: result 中 stream: true

    9. test_standard_params: temperature/top_p/stop_sequences 映射
       - 输入: { temperature:0.7, top_p:0.9, stop_sequences:["END"] }
       - 期望: { temperature:0.7, top_p:0.9, stop:["END"] }

    10. test_thinking_blocks_dropped: thinking blocks 静默丢弃
        - 输入: assistant content 包含 {type:"thinking",thinking:"..."}
        - 期望: input 中该 block 被忽略

    11. test_image_content_to_input_image: 图片内容转换
        - 输入: user content 包含 {type:"image",source:{type:"base64",media_type:"image/png",data:"abc123"}}
        - 期望: input 中转换为 {type:"input_image", image_url:"data:image/png;base64,abc123"}（或按 Responses API 规范转换）

    12. test_clean_schema_applied: JSON Schema 清理
        - 输入: tools 的 input_schema 包含 format/default 字段
        - 期望: 输出的 tools parameters 中 format/default 已被移除
  </behavior>

    2. responses_request.rs 实现：

       - thinking blocks 静默过滤
