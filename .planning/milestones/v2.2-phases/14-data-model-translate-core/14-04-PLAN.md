<feature>
  <name>流式 SSE 状态机 create_anthropic_sse_stream()</name>
  <files>src-tauri/src/proxy/translate/stream.rs</files>
  <behavior>
    --- SSE 行解析 ---
    - "data: {json}\n\n" => 提取 json 部分
    - "data: [DONE]\n\n" => 流结束信号
    - 跨 chunk 截断：不完整 "data: " 行缓冲到下一 chunk 拼接
    - 空行和非 "data:" 前缀行忽略

    --- 文本 delta 事件序列 (STRM-01) ---
    - 输入：OpenAI chunks 含 choices[0].delta.content 文本片段
    - 首个文本 delta 前输出：message_start 事件 + content_block_start(type:text, index:0) 事件
    - 每个文本 delta 输出：content_block_delta(type:text_delta, text:片段) 事件
    - message_start 事件仅输出一次（首个有效 delta 时）
    - message_start 包含完整 Anthropic message 元数据（id, type, role, model, usage 占位）

    --- 工具调用流式转换 + Deferred Start (STRM-02) ---
    - 输入：OpenAI chunks 含 choices[0].delta.tool_calls 数组
    - 每个 tool_call 有 index 字段标识工具编号
    - 首次收到 tool_call 的 id 和 function.name 时才发 content_block_start(type:tool_use)
    - id 或 name 可能分布在不同 chunk：第一个 chunk 可能只有 index + id，下一个才有 name
    - id/name 未就绪时将 function.arguments delta 缓冲到 pending_args
    - id/name 就绪后：先发 content_block_start，再发缓冲的 pending_args 作为 input_json_delta，再发当前 chunk 的 immediate delta
    - 工具 content_block_start 的 index 从文本 block 之后开始编号（如文本是 index 0，第一个工具是 index 1）

    --- 多工具并发 (STRM-03) ---
    - 按 OpenAI tool_calls 的 index 字段独立追踪每个工具的状态
    - 使用 HashMap<usize, ToolBlockState> 管理
    - 不同 index 的工具各自独立的 started/pending_args 状态
    - ToolBlockState 结构体：anthropic_index(u32), id(String), name(String), started(bool), pending_args(String)

    --- 流结束事件 (STRM-04) ---
    - 收到 choices[0].finish_reason 时：
      1. 关闭所有已打开的 content block（发 content_block_stop）
      2. 发 message_delta（含 stop_reason 通过 map_finish_reason 映射，含 usage 如果 chunk 中有）
      3. 发 message_stop
    - 收到 "data: [DONE]" 时：如果 finish_reason 尚未处理，补发 message_stop
    - 所有 SSE 事件格式：`event: {type}\ndata: {json}\n\n`

    --- 错误处理 ---
    - 上游流 chunk 读取错误 => yield IO error，终止流
    - JSON 解析错误 => 跳过该事件，继续处理后续
  </behavior>
