# 设计文档：移除监控页与设置页冗余页标题

**日期：** 2026-03-20

## 背景与动机

当前应用已经通过顶部导航条在 `主页 / 监控 / 设置` 三个主视图之间切换，并且导航项已经展示了明确文字。

在此基础上，监控页与设置页内容区左上角仍各自保留了一份页面大标题，导致同一层级的信息在视觉上重复出现：

- 顶部导航已说明当前所在页面
- 页面内容区标题再次重复“监控”或“设置”

这种重复会让顶部区域显得偏重，分散用户对实际内容与二级标签的注意力。

## 目标

- 移除监控页左上角页标题
- 移除设置页左上角页标题
- 保留两页现有 Tabs 结构与交互行为
- 顺手压缩顶部留白与间距，让页面更紧凑、干净
- 保持两页顶部布局风格一致

## 非目标

- 不调整顶部主导航的结构、文案或交互
- 不修改主页布局
- 不改动监控页与设置页的业务逻辑、状态管理或标签切换逻辑
- 不新增新的导航层级或面包屑

## 现状

### 监控页

`src/components/traffic/TrafficPage.tsx` 当前顶部区域为“页面标题 + Tabs”横向排列：

- 页标题：`t("traffic.title")`
- Tabs：`logs / stats`

### 设置页

`src/components/settings/SettingsPage.tsx` 当前顶部区域为“页面标题 + Tabs”横向排列：

- 页标题：`t("settings.title")`
- Tabs：`general / advanced / about`

### 顶部主导航

`src/components/layout/Header.tsx` 已提供主视图级别的带文字导航，因此监控页与设置页的大标题不再承担“告诉用户当前在哪一页”的主要职责。

## 方案比较

### 方案 A：仅移除页标题，并将 Tabs 区域收紧为紧凑顶栏（推荐）

做法：

- 删除监控页与设置页顶部的 `h2` 标题
- 保留现有 Tabs
- 将顶部容器改为仅承载 Tabs 的紧凑布局
- 略微减少顶部 `padding` 与 `gap`

优点：

- 改动最小
- 风险低
- 与当前视觉结构连续
- 能直接解决“信息重复”和“顶部偏重”问题

缺点：

- 主要是减法优化，不属于大的视觉升级

### 方案 B：移除页标题，并重做更明显的分区条

做法：

- 删除标题
- 为 Tabs 区域增加更强的分区条样式和固定节奏

优点：

- 视觉规范感更强

缺点：

- 超出本次“去冗余 + 微调”的需求范围
- 会引入更多样式决策

### 方案 C：移除页标题，并将 Tabs 完全并入内容区

做法：

- 删除标题
- 进一步压缩顶部独立区域，让 Tabs 更贴近主体内容

优点：

- 最省垂直空间

缺点：

- 容易让页面顶部显得过紧
- 监控页信息密度较高时可读性可能变差

## 最终设计

采用 **方案 A**。

### 布局设计

#### 监控页

将顶部区域从：

- 页面标题 `traffic.title`
- `logs / stats` Tabs

调整为：

- 仅保留 `logs / stats` Tabs
- Tabs 左对齐
- 缩小顶部区域的上下内边距与元素间距

#### 设置页

将顶部区域从：

- 页面标题 `settings.title`
- `general / advanced / about` Tabs

调整为：

- 仅保留 `general / advanced / about` Tabs
- Tabs 左对齐
- 与监控页采用一致的顶部节奏

### 样式原则

- 维持当前组件体系，不引入新的抽象组件
- 仅做局部类名调整
- 统一两页顶部 spacing，避免一页松一页紧
- 保持现有 Tabs 的视觉优先级，不额外增强或弱化

## 影响范围

- `src/components/traffic/TrafficPage.tsx`
- `src/components/settings/SettingsPage.tsx`

不涉及：

- `src/components/layout/Header.tsx`
- 国际化文案资源
- 业务数据流与 hooks

## 数据流与交互

本次改动仅影响展示层，不改变任何状态流：

- 监控页的 `Tabs` 默认值、切换行为、筛选逻辑保持不变
- 设置页的 `Tabs` 默认值与各分区内容保持不变
- 主导航的切页逻辑保持不变

## 错误处理

无需新增错误处理。本次为纯 UI 结构与 spacing 微调，不引入新的异步逻辑、输入校验或异常分支。

## 测试与验证策略

### 手动验证

1. 打开监控页，确认左上角不再显示单独页标题，仅保留 Tabs
2. 打开设置页，确认左上角不再显示单独页标题，仅保留 Tabs
3. 确认两页顶部留白明显收紧，但不拥挤
4. 确认 Tabs 切换功能不受影响
5. 确认主导航仍能清晰表达当前所在页面

### 自动化验证

若已有相关组件测试或可运行前端测试，则验证：

- 页面正常渲染
- Tabs 仍可切换对应内容
- 无因删除标题导致的测试断言失效

## 验收标准

- [ ] 监控页左上角页标题已移除
- [ ] 设置页左上角页标题已移除
- [ ] 两页顶部 Tabs 仍可正常使用
- [ ] 两页顶部 spacing 已统一并收紧
- [ ] 页面没有出现新的视觉层级混乱或明显空白问题

## 备注

本次调整的核心原则是：主视图名称交给顶部主导航表达，页面内容区顶部只保留对当前内容真正有帮助的二级切换元素。
